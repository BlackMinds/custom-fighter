# 《自由格斗 · Custom Fighter》游戏策划文档

| 项目信息 | 内容 |
| --- | --- |
| 游戏名称 | 自由格斗（Custom Fighter） |
| 游戏类型 | 2D 横版格斗（类拳皇/街霸） |
| 运行平台 | 浏览器（Nuxt 3 + Vue 3 + Canvas），可部署至任意 Node 服务器 |
| 当前版本 | V0.1.1（本地双人 + 人机对战 + 指令招式） |
| 文档版本 | 1.1（2026-06-13） |

---

## 1. 核心概念

### 1.1 一句话定位

> 一款"**招式不绑定角色**"的格斗游戏：开打前从公共招式库自由挑选 4 个招式组成自己的战斗风格，再进场对决。

### 1.2 与传统格斗游戏（拳皇）的差异

| 传统拳皇 | 自由格斗 |
| --- | --- |
| 每个角色固定招式表 | **招式库公开，任意组合** |
| 选角色 = 选招式 | 选招式 = 创造自己的"角色" |
| 搓招指令（↓↘→+拳） | 一键技能槽（U/I/O/P）+ 简化指令招式（方向+拳/踢），上手即玩 |
| 指令绑定固定招式 | **指令位（下+拳 / 前+拳 / 下前+拳 / 后+拳 / 后+踢）装什么招式也由你定** |
| 角色性能不对称 | 双方从同一招式池构筑，策略对抗 |

### 1.3 核心乐趣

1. **构筑乐趣**：远程流（双飞行道具+聚气）、突进压制流（双突进+连拳）、防守反击流（升龙+投技）……组合产生流派。
2. **博弈乐趣**：你看得到对手的技能槽，但不知道他什么时候按；构筑相克 + 临场反应双层博弈。
3. **打击感**：打击停顿、画面震动、粒子飞溅、击倒演出，操作反馈即时直接。

---

## 2. 核心玩法循环

```
主菜单 → 选择模式（双人 / 人机）
   ↓
P1 装配 4 个招式 →（双人模式则 P2 也装配 / 人机模式 AI 随机装配）
   ↓
战斗（三局两胜，每局 60 秒）
   ↓
结算（再来一局 / 返回菜单重新构筑）
```

单局循环：**立回拉距离 → 抓硬直出招 → 命中攒能量 → 能量解锁必杀 → KO**。

## 3. 招式装配系统

### 3.1 规则

- 每位玩家自带 2 个基础攻击（直拳、前踢），**不占技能槽**。
- 从技能槽招式库（当前 14 个）中**自由挑选 4 个**装入技能槽，同一招式不可重复装配。
- 技能槽与按键一一对应（P1：U/I/O/P；P2：1/2/3/4 或小键盘 4/5/6/8）。
- **指令位（可选）**：另有 5 个指令位，以"方向+拳/踢"触发（类拳皇指令的简化版）。指令位**不与技能槽共用招式库**，而是**每个指令位有自己的专属招式池（各 2 个候选，二选一）**，互不重复：
  - `下+拳`（P1：S+J）：地裂拳 / 寸劲崩拳
  - `前+拳`（D+J）：碎岩肘 / 真空刃
  - `下前+拳`（S+D+J）：燕返斩 / 螺旋升拳
  - `后+拳`（A+J）：铁山靠 / 拨云掌
  - `后+踢`（A+K）：旋月踢 / 千鸟绊
  - "前/后"相对角色面向，换边自动镜像；方向要求多的指令优先匹配（下前+拳 优先于 下+拳）
  - 指令位可空着不装；未装配时方向+拳/踢退化为基础攻击
- 提供"🎲 随机配置"按钮，一键随机 4 个技能槽 + 5 个指令位（指令位各自从专属池抽取）。
- 人机模式下 AI 的技能槽与指令位均为随机装配。

### 3.2 招式分类

| 类别 | 定位 | 代表招式 |
| --- | --- | --- |
| 打击 | 中近距离主力输出 | 疾风连拳、崩山重锤、升龙击 |
| 突进 | 拉近距离、惩罚远程 | 幻影突袭、流星飞踢 |
| 飞行道具 | 远程压制、控制空间 | 波动弹、气功炮 |
| 投技 | 破防御，近身博弈 | 鬼手擒拿 |
| 辅助 | 资源运营 | 聚气术 |
| 必杀 | 耗能量的高回报招式 | 雷光连斩、觉醒·灭世爆 |

## 4. 战斗系统

### 4.1 基础参数（与代码实现一致）

| 参数 | 数值 |
| --- | --- |
| 逻辑帧率 | 60 帧/秒（固定步长，渲染与逻辑分离） |
| 生命值 | 100 |
| 能量上限 | 100（**跨回合保留**） |
| 移动速度 | 3.2 px/帧 |
| 跳跃初速 / 重力 | 11.5 / 0.6 px/帧² |
| 场地范围 | x ∈ [50, 910]（画面宽 960） |
| 角色推挤间距 | 最小 34 px（防止重叠穿身） |

### 4.2 招式三段帧结构

每个招式由 **前摇（startup）→ 判定（active）→ 后摇（recovery）** 三段组成：

- **前摇**：出招准备，无判定，可被打断（升龙击/灭世爆前摇无敌是例外）。
- **判定**：攻击框生效；多段招式按 `rehitInterval` 间隔重复判定；突进招在此阶段每帧位移。
- **后摇**：收招硬直，是对手反击的窗口。重招后摇大，被防住风险高。

### 4.3 防御系统

- **操作**：按住后方向键（远离对手方向）即防御，无独立防御键。
- **削血（Chip Damage）**：防御成功仍受 15% 伤害（最低 1 点）。
- **防御硬直**：受击硬直 × 0.7 + 4 帧，明显短于被打硬直。
- **投技无视防御**：鬼手擒拿是防御流的天敌，构成"打击 ↔ 防御 ↔ 投技"三角克制。
- 空中、出招中、硬直中无法防御。

### 4.4 能量系统

| 行为 | 能量变化 |
| --- | --- |
| 攻击命中 | +招式 meterGain（4~12） |
| 攻击被防 | +2 |
| 被打中 | +4 |
| 防御住攻击 | +2 |
| 聚气术 | 一次性 +25（动作全程 54 帧，破绽大） |

消耗端：雷光连斩 20 / 气功炮 30 / 觉醒·灭世爆 100。能量不足时技能槽在 HUD 中置灰。

### 4.5 击倒与起身

- 带 `knockdown` 属性的招式命中后对手倒地 55 帧，随后起身 18 帧。
- **倒地与起身全程无敌**，防止无限压制，给翻盘喘息窗口。

### 4.6 飞行道具

- 从角色身前 34px 处生成，直线飞行，命中、抵消或飞出屏幕即消失。
- **敌我双方的飞行道具碰撞会互相抵消**（爆出白色粒子），形成对波博弈。
- 波动弹：速度 7，存在 120 帧；气功炮：速度 4 但又大又痛，存在 180 帧。

### 4.7 打击反馈（手感设计）

| 反馈 | 触发 | 强度 |
| --- | --- | --- |
| 打击停顿（Hitstop） | 命中 | 普通 6 帧 / 重击（伤害≥14）10 帧 / 被防 3 帧 |
| 画面震动 | 命中 | 5 + 伤害×0.4（上限 14） |
| 粒子飞溅 | 命中 | 6 + 伤害 个（上限 24），颜色取招式特效色 |
| 受击白闪 | 受击硬直中 | 角色每 4 帧闪白 2 帧 |
| 满气光环 | 能量=100 | 角色周身金色脉动光圈 |

### 数值设计原则

- **风险收益守恒**：伤害/收益越高 → 前摇或后摇越大（崩山重锤 16 伤但 18 帧前摇；直拳 4 帧出手但只有 5 伤）。
- **每个招式有明确克制对象**：升龙克压制、投技克防御、突进克远程、飞行道具克立回。
- **无纯上位选择**：任意两个招式之间不存在"全方位更强"关系。
- **指令位专属招式各有定位**：同一指令位的两个候选风格相对（如 下+拳 的地裂拳偏快牵制 vs 寸劲崩拳偏爆发；后+拳 的铁山靠偏击退 vs 拨云掌偏跑打），选哪个体现构筑取向。

## 6. 对战规则

| 规则 | 设定 |
| --- | --- |
| 赛制 | 三局两胜（BO3） |
| 单局时间 | 60 秒 |
| 胜负判定 | 先把对手打到 0 血（K.O.)；时间到比剩余血量（TIME UP）；血量相同则平局重赛该局 |
| 回合间 | 血量重置，**能量保留**，位置重置（300 / 660） |
| 开场演出 | ROUND N → FIGHT!（80 帧锁定操作） |

## 7. 操作设计

### 玩家 1（左侧，青色 #4cc9f0）

| 操作 | 按键 |
| --- | --- |
| 移动 / 跳 / 蹲 | A · D / W / S |
| 拳 / 踢 | J / K |
| 技能槽 1-4 | U / I / O / P |
| 指令招式 | S+J / D+J / S+D+J / A+J / A+K（面向右时；面向左 A/D 互换） |
| 防御 | 按住远离对手的方向键 |

### 玩家 2（右侧，品红 #f72585）

| 操作 | 按键 |
| --- | --- |
| 移动 / 跳 / 蹲 | ← · → / ↑ / ↓ |
| 拳 / 踢 | 小键盘1 或 逗号 / 小键盘2 或 句号 |
| 技能槽 1-4 | 数字 1/2/3/4 或 小键盘 4/5/6/8 |
| 指令招式 | ↓+拳 / →+拳 / ↓→+拳 / ←+拳 / ←+踢（前/后随面向镜像） |

设计要点：**放弃复杂搓招**，技能直接绑键 + 简化指令（单方向+按键，无需快速输入序列）。门槛低、双人同键盘不打架，且为未来手柄/触屏适配留余地。方向键为"按住"判定，攻击键为"按下边沿"判定（防止按住连发）。指令匹配规则：按下拳/踢瞬间检查当前按住的方向，方向要求多者优先；"后+拳/踢"与防御不冲突（防御仅在受击瞬间结算）。

## 8. AI 行为设计（人机模式）

AI 每帧产出与玩家相同结构的虚拟输入，按**距离分层决策**，决策间有 6~40 帧冷却模拟人类反应：

| 距离 | 行为权重 |
| --- | --- |
| 远距离（>260px） | 40% 放飞行道具 / 能量<70 时 55% 聚气 / 否则前进 |
| 中距离（120~260px） | 35% 突进技 / 85% 走近 / 偶尔跳入 |
| 近距离（<120px） | 20% 拳 / 18% 踢 / 37% 随机技能或指令招式（装配了指令位时约 45% 概率改用方向+拳/踢 触发） / 13% 防御 / 12% 后撤 |
| 对手出招时（<150px） | 45% 概率应对：小概率升龙反击，否则持续防御 24~40 帧 |
| 满能量且近身 | 50% 概率直接放觉醒必杀 |

后续可扩展难度分级（决策冷却时间、防御概率、确反意识）。

## 9. 界面与流程

### 9.1 屏幕流转

`主菜单` → `P1 选招` →（PVP：`P2 选招`）→ `战斗` → `结算浮层` →（再来一局 / 返回菜单）

### 9.2 各界面要素

- **主菜单**：游戏标题（渐变字）、两个模式按钮、双方键位说明、规则速览。
- **选招界面**：4 个技能槽（显示绑定按键）+ 5 个指令位下拉框（每个下拉只列出该指令位的 2 个专属招式 + "不装"，附伤害/耗气）、14 张技能槽招式卡（名称/类别/描述/伤害/前摇/耗气，选中金框高亮）、随机配置按钮（同时随机技能槽与指令位）、确认出战按钮（不足 4 个时禁用并提示差几个）。
- **战斗 HUD**：顶部双血条（向中央消减）+ 回合胜利点 + 居中倒计时（≤10 秒变红）；底部双能量条（满气金色发光显示 MAX）+ 4 格技能槽提示（按键+招式名，能量不足置灰）+ 技能槽上方一排指令位提示（指令简称+招式名）。
- **演出横幅**：ROUND N / FIGHT! / K.O. / TIME UP / DRAW / XX 获胜，大字描边+弹出动画。

## 10. 美术与表现

- **角色**：程序绘制火柴人骨骼（11 关节：头/颈/髋/双臂肘手/双腿膝脚），每个招式阶段对应一个关键姿势，帧间插值平滑过渡；前后肢体深浅双色区分；行走时腿部正弦摆动。
- **场景**：夜晚都市天台风——渐变夜空、月亮、固定种子生成的楼群剪影与零星灯光、紫色调地面。
- **特效**：招式判定阶段武器端发光（颜色=招式特效色）、飞行道具径向渐变光球+拖尾、命中粒子受重力飞溅、防御出现蓝色护盾弧。
- **色彩规范**：P1 青 `#4cc9f0`、P2 品红 `#f72585`、能量金 `#ffd166`，每个招式有专属特效色（见 `game/moves.ts`）。

## 11. 技术架构

```
KOF/
├─ nuxt.config.ts          # Nuxt 3 配置（SSR 开启）
├─ app.vue                 # 根组件
├─ pages/index.vue         # 屏幕流转状态机（菜单/选招/战斗）
├─ components/
│  ├─ LoadoutSelect.vue    # 招式装配界面（Vue 3）

   ├─ types.ts             # MoveDef / InputFrame / Loadout 等类型
   ├─ moves.ts             # ★ 招式数值库（策划调参主战场）
   ├─ commands.ts          # 指令招式定义（方向+拳/踢 的 5 个指令位）
   ├─ poses.ts             # 火柴人骨骼姿势库
   ├─ fighter.ts           # 角色状态机（移动/出招/受击/倒地）
   ├─ engine.ts            # 战斗引擎（判定/能量/回合/飞行道具/粒子）
   ├─ render.ts            # Canvas 渲染（角色/场景/HUD/横幅）
   ├─ input.ts             # 键盘输入（按住/边沿检测）
   └─ ai.ts                # 人机 AI 决策
```

关键设计决策：

1. **逻辑与渲染分离**：引擎按 60fps 固定逻辑帧推进（`engine.step(inputs)` 纯输入驱动），渲染走 rAF——这是未来**在线对战帧同步/回滚（rollback）的前置条件**。
2. **数据驱动招式**：新增招式只需在 `moves.ts` 加一条数据（帧数/判定框/属性/姿势引用），无需改引擎代码。
3. **Nuxt 3 全栈**：当前仅用前端渲染；Nitro 服务端为在线对战（WebSocket 房间）预留。部署：`npm run build` → `node .output/server/index.mjs`。

## 13. 平衡性风险与对策

| 风险 | 对策 |
| --- | --- |
| 某套构筑无解（如双飞行道具龟缩） | 突进招收益保底；后续点数系统让强招占高分 |
| 投技泛滥 | 投技距离短（判定框 26px）、后摇 20 帧，空跳可躲 |
| 满气必杀一波翻盘感过强 | 100 能量获取需 10+ 次有效命中，且无敌仅覆盖前摇，挥空后摇 30 帧可惩罚 |
| 拖时间龟血 | 削血机制 + 60 秒倒计时按血量判定，逼迫劣势方主动 |
